# 企业光盘刻录系统

## 概述

面向政府机构、银行等受监管行业的企业级Android光盘刻录解决方案。支持法规要求的长期数据归档，确保数据完整性、不可篡改性和完整的审计追踪。

## 核心特性

### 🔒 数据完整性保障
- **DAO模式刻录** (Disc At Once) - 一次性完整写入，避免多会话追加导致的数据碎片化
- **SHA256自动校验** - 刻录完成后自动读取光盘数据，与源文件进行哈希比对
- **扇区级校验** - 校验失败精确定位到具体扇区

### 📋 审计合规
- **完整审计日志** - 记录所有操作事件、时间戳、哈希值
- **结构化日志格式** - 支持自动化日志分析和归档
- **一键导出** - ZIP格式导出，便于审计上报

### 🔌 硬件兼容性
- **USB Host模式** - 支持标准USB外置刻录机
- **即插即用** - 自动检测设备连接
- **多芯片支持** - 覆盖主流USB-SATA桥接芯片

## 快速开始

### 环境要求

- Android Studio Hedgehog (2023.1.1) 或更高版本
- JDK 17
- Android SDK 34
- Android设备：Android 10+ (API 29+)
- 硬件：支持USB Host功能的设备

### 构建步骤

```bash
# 1. 克隆项目
git clone <repository-url>
cd DiscBurnerAndroid

# 2. 打开项目
# 使用Android Studio打开 DiscBurnerAndroid 目录

# 3. 同步Gradle
./gradlew sync

# 4. 构建APK
./gradlew assembleDebug

# 5. 安装到设备
adb install app/build/outputs/apk/debug/app-debug.apk
```

### 首次使用

1. **授予权限**
   - 连接USB刻录机时，系统会弹出权限请求
   - 进入系统设置 → 应用 → 企业光盘刻录 → 权限
   - 开启"所有文件访问权限"（Android 11+需要）

2. **放置ISO文件**
   - 将ISO文件放入设备存储的以下任一目录：
     - `/Backups/`
     - `/ISOs/`
     - `/Downloads/`
     - 应用私有目录

3. **开始刻录**
   - 连接USB刻录机
   - 应用自动检测到设备后，选择ISO文件
   - 点击"开始刻录"
   - 等待刻录和校验完成

## 刻录流程详解

### DAO模式流程

```
1. 设备准备
   ├── 检查USB连接
   ├── 发送TEST UNIT READY
   └── 设置DAO模式 (Mode Select)

2. 光盘检查
   ├── 读取光盘信息 (READ DISC INFORMATION)
   ├── 验证空白状态
   └── 检查可用空间

3. 预留轨道
   └── RESERVE TRACK (一次性预留全部空间)

4. 数据写入
   ├── 流式读取ISO文件
   ├── 计算SHA256哈希
   └── WRITE 10 (64KB批次写入)

5. 关闭光盘
   ├── SYNCHRONIZE CACHE
   ├── CLOSE TRACK
   └── CLOSE SESSION (Finalization)

6. 数据校验
   ├── READ 10 (读取光盘数据)
   ├── 计算SHA256哈希
   └── 比对源文件与光盘哈希
```

## 故障排除

### 常见问题

**Q: 应用无法检测到刻录机**
- 确认设备支持USB Host (OTG)
- 使用标准Type-C转USB-A转接头
- 检查刻录机是否正常供电（部分高功率设备需要外接电源）

**Q: 刻录失败，错误码A006 (WRITE_ERROR)**
- 光盘质量不佳，更换品牌光盘
- 刻录机缓存不足，降低写入速度（需修改代码）
- USB连接不稳定，更换数据线

**Q: 校验失败 (VERIFY_MISMATCH)**
- **这是严重的数据完整性问题**
- 光盘质量差或已损坏
- 刻录机激光头老化
- 立即停止使用该光盘和刻录机，更换设备

**Q: Android 14无法访问外部存储**
- 进入系统设置手动授予"所有文件访问权限"
- 或使用应用私有目录（Android/data/com.enterprise.discburner/files/）

## 安全注意事项

1. **数据安全**
   - 刻录完成后务必确认校验通过
   - 校验失败的光盘可能包含错误数据，不可用于备份
   - 敏感数据刻录后应物理销毁光盘

2. **审计合规**
   - 定期导出并归档审计日志
   - 日志文件包含敏感路径信息，妥善保管
   - 建议将日志同步到安全的中央服务器

3. **硬件安全**
   - 只使用可信来源的刻录机和光盘
   - 定期检查刻录机状态，老化设备及时更换
