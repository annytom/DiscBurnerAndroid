# 企业光盘刻录系统 V2.0

面向政府机构、银行等受监管行业的企业级Android光盘刻录解决方案。

## ✨ 核心特性

### 1. 任意文件刻录
- **文件选择** - 支持选择任意格式的文件和文件夹
- **自动打包** - 自动生成ISO 9660标准文件系统
- **卷标设置** - 自定义光盘卷标

### 2. 多会话（补刻）支持
- **追加刻录** - 在已有数据的光盘上追加新文件
- **会话管理** - 支持多轨道、多会话管理
- **写入模式** - DAO/TAO/SAO三种模式灵活选择

### 3. 数据完整性保障
- **DAO/TAO/SAO模式** - 根据场景选择最佳刻录模式
- **SHA256自动校验** - 刻录后自动校验数据完整性
- **扇区级验证** - 精确定位到具体扇区

### 4. 审计合规
- **完整审计日志** - 记录所有操作事件、时间戳、哈希值
- **会话追踪** - 多会话刻录的完整链路记录
- **一键导出** - ZIP格式导出，便于审计上报

## 📱 使用流程

### 首次刻录（空白光盘）

1. 连接USB刻录机
2. 点击"分析光盘"确认光盘状态
3. 选择"刻录文件"或"刻录ISO"标签页
4. 添加要刻录的文件或选择ISO镜像
5. 设置卷标（文件刻录时）
6. 选择写入模式（推荐DAO）
7. 选择是否关闭光盘
8. 点击"开始刻录"

### 追加刻录（补刻）

1. 连接刻录机，放入已刻录但未关闭的光盘
2. 点击"分析光盘"查看现有会话
3. 系统自动检测可追加状态
4. 选择"TAO模式"（追加必须）
5. **不要勾选"关闭光盘"**（如需继续追加）
6. 添加新文件
7. 点击"开始刻录"
8. 新数据将追加为新会话

## 🔧 写入模式说明

| 模式 | 说明 | 适用场景 | 可追加 |
|------|------|----------|--------|
| **DAO** | 整盘刻录（Disc At Once） | 空白盘、要求高完整性 | ❌ 否 |
| **TAO** | 轨道刻录（Track At Once） | 追加刻录、分段写入 | ✅ 是 |
| **SAO** | 会话刻录（Session At Once） | 多会话管理 | ✅ 是 |

### 模式选择建议

- **空白光盘**：使用DAO模式，数据完整性最高
- **追加数据**：使用TAO模式，必须光盘未关闭
- **多版本备份**：使用SAO模式，每版本一个会话

## 📂 ISO生成说明

当选择"刻录文件"时，应用会自动：

1. 扫描所选文件/目录
2. 计算所需光盘空间
3. 生成ISO 9660 Level 2文件系统
4. 自动处理文件名（8.3格式转换）
5. 生成Joliet扩展（支持长文件名）
6. 刻录完成后自动删除临时ISO

## 🔄 多会话工作原理

```
光盘布局（多会话示例）:
┌─────────────────────────────────────────────────────┐
│ 导入区 (Lead-in)                                     │
├─────────────────────────────────────────────────────┤
│ 会话1 卷描述符                                        │
│ 会话1 路径表                                          │
│ 会话1 目录记录                                        │
│ 会话1 文件数据 (Track 1)                              │
├─────────────────────────────────────────────────────┤
│ 会话2 卷描述符（指向会话1）                              │
│ 会话2 路径表                                          │
│ 会话2 目录记录                                        │
│ 会话2 文件数据 (Track 2)  ← 追加的数据在这里            │
├─────────────────────────────────────────────────────┤
│ 导出区 (Lead-out)                                    │
└─────────────────────────────────────────────────────┘
```

### 会话关闭选项

- **不关闭会话**：当前会话可继续追加（仅限本次追加）
- **关闭会话**：当前会话完成，可开启新会话追加
- **关闭光盘**：光盘最终化，不可再追加

## 🛡️ 安全注意事项

1. **追加刻录风险**
   - 多会话光盘的兼容性较单会话差
   - 老旧光驱可能无法读取追加的数据
   - 建议重要数据使用单会话DAO模式

2. **数据验证**
   - 每次刻录（包括追加）都会进行SHA256校验
   - 校验失败的光盘数据不可信
   - 追加刻录不影响已有数据的校验

3. **审计追踪**
   - 每次会话都有独立的会话ID
   - 日志记录完整的追加链路
   - 可追溯每个文件的刻录历史

## 🐛 故障排除

### 无法追加刻录

**症状**："光盘已关闭"或"不可追加"

**解决**：
- 确认光盘未在之前的刻录中勾选"关闭光盘"
- 使用TAO或SAO模式（DAO不支持追加）
- 检查光盘物理状态（划痕、污渍）

### 追加后旧数据不可见

**症状**：只能看到新追加的文件

**解决**：
- 这是某些老旧操作系统的已知问题
- 在Windows 10+/macOS/Linux上测试
- 建议使用ISO编辑软件检查多会话结构

### ISO生成失败

**症状**："ISO生成失败"

**解决**：
- 检查存储空间（需要2倍于数据大小的临时空间）
- 检查文件名是否包含非法字符
- 尝试减少文件数量或总大小

## 📊 技术规格

### 支持的文件系统
- ISO 9660 Level 1/2
- Joliet扩展（Unicode文件名）
- Rock Ridge（Unix权限，计划中）

### 多会话限制
- 最大会话数：99（光盘限制）
- 最大轨道数：99（会话限制）
- 每个会话最小空间：几MB（卷描述符开销）

### 兼容性
- Windows：完全支持（需要支持多会话的驱动）
- macOS：完全支持
- Linux：完全支持
- DVD播放器：通常只读取第一个会话
