# 企业光盘刻录系统 V2.0.0

面向政府机构、银行等受监管行业的企业级Android光盘刻录解决方案。

**状态**: 阶段2完成 ✅ | **最后更新**: 2026-03-27

## ✨ 核心特性

### 1. 任意文件刻录
- **文件选择** - 支持选择任意格式的文件和文件夹
- **自动打包** - 自动生成ISO 9660标准文件系统
- **卷标设置** - 自定义光盘卷标
- **刻录速度** - 支持4x/8x/16x/24x/32x/48x/52x速度选择

### 2. 刻录机型号选择
- **自动检测** - 连接设备后自动识别刻录机型号
- **手动选择** - 从型号数据库中选择匹配的刻录机
- **品牌支持** - ASUS、LG、Samsung、Lite-On、Pioneer、Sony、HP等主流品牌
- **智能配置** - 根据型号自动配置最大速度、缓冲区大小等参数
- **特性识别** - 支持防刻死、双层刻录、光雕等特性显示

### 3. 多会话（补刻）支持
- **追加刻录** - 在已有数据的光盘上追加新文件
- **会话管理** - 支持多轨道、多会话管理
- **写入模式** - DAO/TAO/SAO三种模式灵活选择

### 3. 刻录队列管理
- **批量任务** - 一次性添加多个刻录任务
- **优先级管理** - 低/普通/高/紧急四个优先级
- **自动处理** - 可选自动连续处理队列
- **失败重试** - 自动重试失败任务（最多2次）
- **队列状态** - 实时显示待处理/进行中/已完成/失败任务

### 4. 审计合规（增强版）
- **完整审计日志** - 记录所有操作事件、时间戳、哈希值
- **数字签名** - HMAC-SHA256防篡改签名验证
- **双存储模式** - 文件（兼容）+ Room数据库（查询）
- **会话追踪** - 多会话刻录的完整链路记录
- **历史记录** - 查看过往刻录会话和统计
- **一键导出** - ZIP/JSON格式导出，便于审计上报

### 5. 数据完整性保障
- **DAO/TAO/SAO模式** - 根据场景选择最佳刻录模式
- **SHA256自动校验** - 刻录后自动校验数据完整性
- **SCSI重试机制** - 指数退避策略，自动重试临时错误
- **扇区级验证** - 精确定位到具体扇区

## 📱 使用流程

### 首次刻录（空白光盘）

1. 连接USB刻录机
2. 点击"分析光盘"确认光盘状态
3. 选择"刻录文件"或"刻录ISO"标签页
4. 添加要刻录的文件或选择ISO镜像
5. 设置卷标（文件刻录时）
6. 选择写入模式（推荐DAO）
7. 选择刻录速度（0=最大速度）
8. 选择是否关闭光盘
9. 点击"开始刻录"

### 追加刻录（补刻）

1. 连接刻录机，放入已刻录但未关闭的光盘
2. 点击"分析光盘"查看现有会话
3. 系统自动检测可追加状态
4. 选择"TAO模式"（追加必须）
5. **不要勾选"关闭光盘"**（如需继续追加）
6. 添加新文件
7. 点击"开始刻录"
8. 新数据将追加为新会话

### 使用队列批量刻录

1. 进入"队列管理"界面
2. 添加多个刻录任务（支持不同文件/ISO）
3. 设置任务优先级（紧急任务优先）
4. 开启"自动处理"模式
5. 系统自动按顺序刻录
6. 查看历史记录界面了解刻录统计

### 选择刻录机型号

1. 连接USB刻录机后，系统自动检测型号
2. 在设备状态卡片查看检测到的型号
3. 点击"更改"按钮进入型号选择界面
4. 可接受自动检测结果，或手动选择准确型号
5. 选择型号后，系统根据型号规格自动配置刻录参数

**型号数据库包含：**
- ASUS: DRW-24B3ST, SDRW-08D2S-U
- LG: GH24NSD1, GP65NB60
- Samsung: SE-208GB, SH-224FB
- Lite-On: iHAS324, eBAU108
- Pioneer: DVR-S21WBK
- Sony: AD-7290H, DRX-S90U
- HP: DVD1270i, DVD556s
- Buffalo: DVSM-PC58U2V
- Transcend: TS8XDVDS

## 🔧 写入模式说明

| 模式 | 说明 | 适用场景 | 可追加 |
|------|------|----------|--------|
| **DAO** | 整盘刻录（Disc At Once） | 空白盘、要求高完整性 | ❌ 否 |
| **TAO** | 轨道刻录（Track At Once） | 追加刻录、分段写入 | ✅ 是 |
| **SAO** | 会话刻录（Session At Once） | 多会话管理 | ✅ 是 |

### 模式选择建议

- **空白光盘**：使用DAO模式，数据完整性最高
- **追加数据**：使用TAO模式，必须光盘未关闭
- **多版本备份**：使用SAO模式，每版本一个会话

## 🚀 刻录速度配置

在`WriteOptions`中配置刻录速度：

```kotlin
data class WriteOptions(
    val writeMode: WriteMode = WriteMode.TAO,
    val writeSpeed: Int = 0,  // 0=最大速度, 4, 8, 16, 24, 32, 40, 48, 52
    val closeSession: Boolean = false,
    val closeDisc: Boolean = false,
    val verifyAfterBurn: Boolean = true,
    val sessionName: String? = null
)
```

**速度建议**：
- **重要数据**：使用较低速度（8x-16x），降低错误率
- **快速备份**：使用最大速度（0=自动选择）
- **老旧光盘**：降低速度以提高成功率

## 📂 ISO生成说明

当选择"刻录文件"时，应用会自动：

1. 扫描所选文件/目录
2. 计算所需光盘空间
3. 生成ISO 9660 Level 2文件系统
4. 自动处理文件名（8.3格式转换）
5. 生成Joliet扩展（支持长文件名）
6. 刻录完成后自动删除临时ISO

## 🔄 多会话工作原理

```
光盘布局（多会话示例）:
┌─────────────────────────────────────────────────────┐
│ 导入区 (Lead-in)                                     │
├─────────────────────────────────────────────────────┤
│ 会话1 卷描述符                                        │
│ 会话1 路径表                                          │
│ 会话1 目录记录                                        │
│ 会话1 文件数据 (Track 1)                              │
├─────────────────────────────────────────────────────┤
│ 会话2 卷描述符（指向会话1）                              │
│ 会话2 路径表                                          │
│ 会话2 目录记录                                        │
│ 会话2 文件数据 (Track 2)  ← 追加的数据在这里            │
├─────────────────────────────────────────────────────┤
│ 导出区 (Lead-out)                                    │
└─────────────────────────────────────────────────────┘
```

### 会话关闭选项

- **不关闭会话**：当前会话可继续追加（仅限本次追加）
- **关闭会话**：当前会话完成，可开启新会话追加
- **关闭光盘**：光盘最终化，不可再追加

## 🛡️ 安全与合规

### 审计日志防篡改

每个审计日志条目都包含HMAC-SHA256数字签名：

```kotlin
// 签名数据格式：sessionId:eventCode:timestamp:details
val signatureData = "$sessionId:$eventCode:$timestamp:$details"
val signature = generateHmacSignature(signatureData, secretKey)
```

**验证机制**：
- 导出时可验证日志完整性
- 篡改的日志条目验证会失败
- 支持数据库+文件双存储验证

### SCSI命令重试机制

自动重试以下可恢复错误：
- UNIT_ATTENTION（设备状态变化）
- NOT_READY（设备未就绪）
- MEDIUM_ERROR（介质错误）
- BUSY（设备忙）

**指数退避策略**：100ms → 200ms → 400ms，最大延迟2秒

### 数据验证

- 每次刻录（包括追加）都会进行SHA256校验
- 校验失败的光盘数据不可信
- 追加刻录不影响已有数据的校验

### 审计追踪

- 每次会话都有独立的会话ID
- 日志记录完整的追加链路
- 可追溯每个文件的刻录历史
- 支持JSON格式批量导出

## 🐛 故障排除

### 无法追加刻录

**症状**："光盘已关闭"或"不可追加"

**解决**：
- 确认光盘未在之前的刻录中勾选"关闭光盘"
- 使用TAO或SAO模式（DAO不支持追加）
- 检查光盘物理状态（划痕、污渍）

### 追加后旧数据不可见

**症状**：只能看到新追加的文件

**解决**：
- 这是某些老旧操作系统的已知问题
- 在Windows 10+/macOS/Linux上测试
- 建议使用ISO编辑软件检查多会话结构

### ISO生成失败

**症状**："ISO生成失败"

**解决**：
- 检查存储空间（需要2倍于数据大小的临时空间）
- 检查文件名是否包含非法字符
- 尝试减少文件数量或总大小

### SCSI命令超时

**症状**：刻录过程中断，显示超时错误

**解决**：
- 检查USB连接是否稳定
- 尝试降低刻录速度
- 重启刻录机后重试

### 审计日志验证失败

**症状**：导出日志时提示签名验证失败

**解决**：
- 检查数据库文件是否被外部修改
- 导出文件日志作为备份
- 重新初始化数据库（会丢失历史记录）

## 📊 技术规格

### 支持的文件系统
- ISO 9660 Level 1/2
- Joliet扩展（Unicode文件名）
- Rock Ridge（Unix权限，计划中）

### 多会话限制
- 最大会话数：99（光盘限制）
- 最大轨道数：99（会话限制）
- 每个会话最小空间：几MB（卷描述符开销）

### 兼容性
- Windows：完全支持（需要支持多会话的驱动）
- macOS：完全支持
- Linux：完全支持
- DVD播放器：通常只读取第一个会话

### 数据库规格
- **引擎**：Room (SQLite)
- **实体**：AuditLog, BurnSession, BurnFile, DeviceInfo
- **容量**：受设备存储限制，建议定期导出清理
